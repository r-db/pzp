IMPLEMENTATION GUIDE

Build your PZP site.

From zero to machine-native in 5 steps.

Quick Start

Create a minimal PZP site in under 10 minutes.

Terminal
# Create directory structure
mkdir -p pzp-site/{context,actions,data,meta}

# Create required files
touch pzp-site/index.md
touch pzp-site/manifest.json
touch pzp-site/context/essential.md
touch pzp-site/robots.txt

Step 1: Create manifest.json

The manifest tells agents what your site is and what it offers.

manifest.json
{
  "$schema": "https://pzp.riscent.com/schemas/manifest.schema.json",
  "version": "1.0.0",
  "protocolVersion": "1.0.0",
  "name": "Your Site Name",
  "description": "One-line description of what this site offers",
  "type": "service",
  "updated": "2026-03-06T00:00:00Z",
  "tokenBudgets": {
    "essential": 500,
    "full": 2000
  },
  "endpoints": {
    "context": "/context/",
    "actions": "/actions/",
    "data": "/data/",
    "meta": "/meta/"
  },
  "capabilities": ["read", "query"],
  "rateLimit": {
    "requests": 100,
    "window": "1m"
  }
}

Step 2: Create index.md

The entry point for agents visiting your site.

index.md
# Your Site Name

> One-line description of what this site offers.

## Identity

- **Domain:** pzp.yourdomain.com
- **Type:** service
- **Version:** 1.0.0
- **Updated:** 2026-03-06

## Purpose

2-3 sentences explaining what agents can do here
and why they would want to.

## Quick Start

1. Read context: [/context/essential.md](context/essential.md)
2. See actions: [/actions/index.json](actions/index.json)
3. Get data: [/data/](data/)

## Navigation

| Resource | Path | Description |
|----------|------|-------------|
| Essential Context | /context/essential.md | Start here |
| Full Context | /context/full.md | Deep dive |
| Actions | /actions/index.json | What you can do |

Step 3: Create context/essential.md

The minimum context an agent needs. Target ~500 tokens.

context/essential.md
# Your Site Essential Context

## What This Is

Single paragraph explaining your site/service.
Keep it concise but complete.

## Key Capabilities

- Capability one
- Capability two
- Capability three

## Primary Actions

| Action | Endpoint | Method | Description |
|--------|----------|--------|-------------|
| search | /api/search | POST | Search content |
| get | /api/item/{id} | GET | Get specific item |

## Constraints

- Rate limited to 100 requests/minute
- Authentication required for write operations
- JSON responses only

Step 4: Create robots.txt

Enable agent discovery of your PZP site.

robots.txt
# Prime Zero Protocol Site
User-agent: *
Allow: /

# PZP Discovery
X-PZP-Site: true
X-PZP-Manifest: /manifest.json
X-PZP-Version: 1.0.0

# AI Agents Welcome
User-agent: GPTBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /

Sitemap: /sitemap.txt

Step 5: Deploy

Set up your subdomain and configure headers.

Deployment Checklist

Required Headers

HTTP Headers
Content-Type: text/markdown; charset=utf-8
X-Token-Count: 450
X-Content-Version: 1.0.0
X-Last-Modified: 2026-03-06T00:00:00Z
X-PZP-Version: 1.0.0
Access-Control-Allow-Origin: *
Cache-Control: public, max-age=3600

Optional: DNS Discovery

DNS TXT Record
_pzp.yourdomain.com TXT "v=pzp1 endpoint=pzp.yourdomain.com type=service"

Optional: HTML Link

On your human site
<link rel="pzp-site" href="https://pzp.yourdomain.com/">

Verification

Test your PZP site after deployment.

Terminal
# Check manifest
curl -s https://pzp.yourdomain.com/manifest.json | jq .

# Check headers
curl -I https://pzp.yourdomain.com/context/essential.md

# Check robots.txt
curl https://pzp.yourdomain.com/robots.txt

# Estimate token count
curl -s https://pzp.yourdomain.com/context/essential.md | wc -w
# Multiply by ~1.3 for approximate tokens

Adding Actions

If your site has APIs, define them in actions/index.json.

actions/index.json
{
  "$schema": "https://pzp.riscent.com/schemas/actions.schema.json",
  "version": "1.0.0",
  "description": "Available actions",
  "baseUrl": "https://api.yourdomain.com",
  "actions": [
    {
      "name": "search",
      "description": "Search the content",
      "endpoint": "/api/search",
      "method": "POST",
      "input": {
        "query": {
          "type": "string",
          "required": true,
          "description": "Search query"
        }
      },
      "output": {
        "type": "array",
        "items": {
          "title": "string",
          "content": "string"
        }
      },
      "errors": {
        "400": "Invalid query",
        "429": "Rate limit exceeded"
      }
    }
  ]
}
